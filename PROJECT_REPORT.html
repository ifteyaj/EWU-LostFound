1. Executive Summary

The EWU Lost & Found system is a comprehensive web-based application designed to facilitate the reporting and recovery of lost and found items within the East West University campus. This platform provides a centralized, secure, and user-friendly solution that connects students who have lost items with those who have found them.

The system implements modern web technologies including PHP, MySQL, JavaScript, and responsive CSS to deliver a seamless user experience across all devices. Key features include user authentication, item reporting, advanced search and filtering, profile management, and real-time item status updates.

2. Project Overview

2.1 Background

Lost and found items on university campuses often go unclaimed due to lack of a centralized reporting system. Students frequently lose valuable items such as ID cards, wallets, mobile phones, and academic materials, while those who find items have no efficient way to return them to their rightful owners.

2.2 Purpose

  • Provide a centralized platform for reporting lost and found items
  • Enable efficient matching between lost and found items
  • Facilitate secure communication between finders and owners
  • Reduce item loss and improve recovery rates on campus
  • Create a trusted community-based recovery system

4. Database Design

4.1 Database Tables

Users Table

Field Type Description
id INT (PK) Primary key, auto-increment
student_id VARCHAR(50) Unique student identifier
email VARCHAR(255) Unique email address
password_hash VARCHAR(255) Bcrypt hashed password
full_name VARCHAR(255) User's full name
is_admin BOOLEAN Admin privilege flag
created_at TIMESTAMP Account creation time

Lost Items & Found Items Tables

Both tables follow a similar structure with the following key fields:

  • id - Primary key
  • user_id - Foreign key to users table
  • item_name - Name of the item
  • category - Item category (Electronics, Documents, etc.)
  • description - Detailed item description
  • location - Last known / found location
  • date_lost/found - Date of incident
  • image - Uploaded image filename
  • status - Item status (pending, claimed, resolved)

5. Key Features and Functionalities

5.1 User Authentication

  • Registration: Email validation (@ewubd.edu), password hashing, student ID verification
  • Login: Secure credential validation, session management, last login tracking
  • Password Recovery: Email-based reset token, secure reset process
  • Logout: Complete session destruction

5.2 Item Reporting

  • Report lost items with detailed descriptions
  • Report found items with location information
  • Image upload support (JPEG, PNG, GIF)
  • Category selection (Electronics, Documents, Personal Items, etc.)
  • Date and location tracking
  • Form validation (client-side and server-side)

5.3 Search & Filtering

  • Keyword-based search across item names and descriptions
  • Filter by category
  • Filter by item type (Lost/Found)
  • Filter by status (Pending/Claimed/Resolved)
  • Combined filter support

5.4 User Profile Management

  • View personal statistics (number of reports)
  • Manage all personal reports
  • Edit and delete own reports
  • Update item status
  • View account information

5.5 Responsive Design

  • Mobile-first approach
  • Breakpoints for tablets and desktops
  • Touch-friendly interface
  • Hamburger menu for mobile navigation
  • Optimized images for all screen sizes

6. Security Features

Implemented Security Measures:

  • Password Security: Bcrypt hashing with automatic salt generation
  • SQL Injection Prevention: Prepared statements for all database queries
  • XSS Prevention: Input sanitization and output encoding with htmlspecialchars()
  • File Upload Security: Type validation, size limits, unique filename generation
  • Session Security: Secure session configuration, timeout management
  • Access Control: Authorization checks for all sensitive operations
  • Environment Configuration: .env file for sensitive credentials

7. Implementation Highlights

7.1 MVC-Like Architecture

The application follows a structured approach with separation of concerns:

  • Views: PHP pages with HTML/CSS for presentation
  • Controllers: Handler files for processing form submissions
  • Models: Database interaction through MySQLi
  • Includes: Reusable components (navbar, footer, auth functions)

7.2 File Structure

Well-organized directory structure:

  • assets/ - CSS, JavaScript, images
  • auth/ - Authentication pages and handlers
  • config/ - Configuration files (database, constants)
  • handlers/ - Form processing scripts
  • includes/ - Reusable components and helper functions
  • uploads/ - User uploaded images
  • errors/ - Custom error pages (404, 403, 500)
